<form>
    - fundamentally serves as a receptacle for diverse input elements enabling users to send data to a web server

    - input elements 
        - text input
            - <input type = "text">
            - allows users to enter single-line text, such as names or email addresses

        - password input
            - <input type = "password">
            - conceals user input for secure data, like passwords

        - checkbox
            - <input type = "checkbox">
            - lets users select multiple options from a list

        - radio button
            - <input type = "radio">
            - provides a list of exclusive choices

        - submit button
            - <input type = "submit">
            - submits the form data to the server

        - textarea
            - <textarea>
            - allows users to input multi-line text, like comments or messages

        - buttons
            - <button>
            - creates custom buttons for form actions, like resetting or submitting

        - label
            - <label>
            - provides a text description for a form element, improving accessibility

        - email input
            - <input type = "email">
            - ensures user input adheres to a valid email format

        - number input
            - <input type = "number">
            - restricts input to numeric values

        - date input
            - <input type = "date">
            - provides date picker for selecting date


    - form attributes

        - action and method - like the gps coordinates for the form data
            - action - specifies the URL where the form data should be sent when the user submits it
                - <form action = "URL">

            - method - determines how the form data is sent to the server
                - GET appends the form data to the URL, POST sends it in the background
                - <form method = "post">

        - name
            - assigned to each form element, allowing you to identify and process the data on the server

            - like labeling each piece of info

        - placeholder
            - provides a hint or example of what's expected in an input field
            - <input placeholder = "placeholder text">

        - autocomplete
            - suggests previously entered values when filling out a form
            - <input type = "text" id = "city" name = "city" autocomplete = "on"

    - form validation and accessibility

        - validation attributes
            - required
                - ensures users must fill out a particular field before submitting

            - pattern
                - allows you to specify a regular eexpressign that the input value must match
                - <input type = "text" id = "phone" name = "phone" pattern = "[0-9]{10}" placeholder = "Enter 10 digit phone number"

            - min and max
                - define the min and max values for numeric inputs, ensuring data falls in that range
                - <input type = "number" id = "age" name = "age" min = "18" max = "100">
